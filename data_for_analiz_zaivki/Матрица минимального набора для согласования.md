# Матрица минимального набора для согласования

Эта таблица нужна для согласования базового набора **полей первой очереди**. Просьба проверить, достаточно ли этих данных для дальнейшей **базовой** аналитики, либо нужно добавить дополнительные поля.

| Поле из минимального набора | Где должно храниться | Для чего используется |
| --- | --- | --- |
| Имя клиента | Лид, Контакт | Для идентификации клиента и дальнейшего создания/обновления контакта. |
| Телефон | Лид, Контакт, связанная Сделка | Для связи с клиентом, дедупликации, обработки звонков и привязки коммуникаций. |
| Email | Лид, Контакт, связанная Сделка | Для переписки, отправки КП, счетов, документов и связи email-заявок с клиентом. |
| Комментарий клиента | Лид, Сделка | Для фиксации потребности клиента, вопроса, задачи или контекста обращения. |
| Тип заявки | Лид, Сделка | Для разделения обращений по каналам: форма, звонок, почта, мессенджер, ручная заявка. |
| Название формы | Лид, Сделка | Для понимания, через какую форму пришла заявка и какие формы дают результат. |
| Сайт / домен | Лид, Сделка | Для аналитики по сайтам и разделения заявок, если источников/доменов несколько. |
| URL страницы заявки | Лид, Сделка | Для понимания, с какой конкретной страницы пришла заявка. |
| Товар / услуга | Лид, Сделка | Для анализа спроса, распределения заявок по направлениям и связи с продажами. |
| Отдел / направление | Лид, Сделка, Воронка | Для маршрутизации заявки, назначения ответственного и аналитики по бизнес-направлениям. |
| Источник CRM | Лид, Сделка | Для фиксации базового источника обращения: реклама, SEO, звонок, почта, мессенджер и т.д. |
| UTM Source | Лид, Сделка | Для определения рекламного или маркетингового источника трафика. |
| UTM Medium | Лид, Сделка | Для определения типа канала: cpc, organic, referral, email и т.д. |
| UTM Campaign | Лид, Сделка | Для анализа эффективности рекламных кампаний. |
| UTM Content | Лид, Сделка | Для анализа объявлений, баннеров, креативов или вариантов ссылок. |
| UTM Term | Лид, Сделка | Для анализа ключевых фраз, условий показа или поискового запроса. |
| yclid | Лид, Сделка | Для связи заявки с конкретным кликом из Яндекс.Директа. |
| ClientID Метрики | Лид, Сделка | Для связи заявки с визитом пользователя в Яндекс.Метрике. |
| Ответственный менеджер | Лид, Сделка | Для контроля обработки заявки и распределения ответственности. |
| Статус обработки заявки | Лид, Сделка | Для понимания текущего этапа обработки заявки или сделки. |
| Результат обращения | Лид, Сделка | Для фиксации итога обращения: квалифицировано, нецелевой, создана сделка, отказ, нет связи. |
| Причина нецелевой заявки | Лид, Сделка | Для анализа качества входящих обращений и рекламного трафика. |
| Сумма сделки | Сделка | Для связи заявок и источников с реальными деньгами и продажами. |
| Причина проигрыша сделки | Сделка | Для анализа, почему сделки не доходят до продажи, и поиска точек улучшения. |

## Вопросы для согласования

1. Достаточно ли этих полей для **базовых** отчетов руководителей?
2. Нужно ли добавить поля, которые важны для конкретных отделов?
